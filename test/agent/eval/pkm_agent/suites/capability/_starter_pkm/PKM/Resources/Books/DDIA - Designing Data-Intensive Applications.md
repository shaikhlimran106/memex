# Designing Data-Intensive Applications (Kleppmann)

Reading notes, ongoing. Roughly one chapter per week.

<!-- fact_id: fact_pkm_books_ddia_001 -->
- Ch 1 — Reliability / scalability / maintainability. The "fault tolerance is not the same as high availability" framing stuck with me.
- Ch 2 — Data models. The relational vs document trade-off is more about query patterns than storage.
- Ch 3 — Storage engines. Log-structured (LSM) trades write speed for compaction cost; B-trees trade insert cost for read locality.
